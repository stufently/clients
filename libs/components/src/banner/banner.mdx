## Content Slots

Banners support three content projection slots.

### Default slot

Plain text or inline elements. Use `bitLink` with `[linkType]` matching the banner `variant` for
inline links.

```html
<bit-banner variant="primary">
  Your message here. <a bitLink linkType="primary">Learn more</a>
</bit-banner>
```

### Title slot (`slot="title"`)

Renders a bold heading above the body text. When a title is present, the actions slot is also
rendered. Apply `slot="title"` to any inline element.

```html
<bit-banner variant="primary">
  <span slot="title">Title text</span>
  Body text here.
</bit-banner>
```

### Actions slot (`slot="actions"`)

Renders below the body text. Only visible when a title is projected. Wrap action elements in
`ng-container slot="actions"`.

```html
<bit-banner variant="primary">
  <span slot="title">Title text</span>
  Body text here.
  <ng-container slot="actions">
    <button bitButton type="button" buttonType="secondary">Cancel</button>
    <button bitButton type="button" buttonType="primary">Continue</button>
  </ng-container>
</bit-banner>
```

## Variants

Use the following guidelines to help choose the correct variant.

### Primary

<Canvas of={stories.Primary} />

Used primarily to encourage the user to upgrade to premium or communicate general information.

### Success

<Canvas of={stories.Success} />

Used to communicate release notes, server maintenance, or other informative events.

### Warning

<Canvas of={stories.Warning} />

Used to alert the user of outdated info or versions.

### Danger

<Canvas of={stories.Danger} />

Rarely used, but may be used to alert users over critical messages or very outdated versions.

## Accessibility

Banners set the `role="status"` and `aria-live="polite"` attributes to ensure screen readers
announce the content prior to the rest of the page. This behavior can be disabled by setting
`[useAlertRole]="false"`.
